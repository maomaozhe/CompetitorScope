# 实施计划 (Implementation Plan)

> 竞品分析 Agent 协作系统的分步实施路线与验收标准。
> 最后更新：2026-05-14（已精简任务表格，保留 Step 框架）

---

## 总策略

**文档先行 → 后端骨架 → 核心管道跑通 → HITL + 并发 → 前端 → 联调 → Demo 打磨**

---

## Step 0 — 文档落盘 ✅

**目标**：把架构与决策固化为文档，作为后续实施的契约。

---

## Step 1 — 项目脚手架 ✅

**目标**：建立可运行的空项目骨架（后端 + 前端）。

---

## Step 2 — 数据模型与基础服务 ✅

**目标**：把架构里的"名词"先建好，后续 Agent 直接装配。

---

## Step 3 — 工具层 ✅

**目标**：Agent 所需的外部能力封装就绪。

---

## Step 4 — 核心管道最小可用版（串行）✅

**目标**：5 个 Agent 节点串行跑通，端到端生成报告。

---

## Step 5 — 并发 fan-out ✅

**目标**：把串行改为并发，Collector × N 和 Analyst × N 同时跑。

---

## Step 6 — HITL 集成 ✅

**目标**：4 个 HITL 介入点全部落地，支持 interactive 和 auto 两种模式。

---

## Step 7 — FastAPI 接口层 ✅

**目标**：REST API 完整可用。

---

## Step 8 — Next.js 前端 ✅

**目标**：Web UI 完整可用，三栏布局 + Agent 进度流 + HITL 弹窗 + 证据下钻。

---

## Step 9 — 端到端联调 + Demo 打磨 🔜

**目标**：前后端联调完成，Demo 场景跑通。

**任务**
1. AI Coding IDE 场景完整跑通
2. 性能优化：确保 5 家竞品端到端 ≤ 5 分钟
3. 兜底方案：预录 trace 回放
4. Prompt 调优
5. 演示模式打磨
6. 错误处理 + 边界场景

**验收**
- Demo 场景 3 分钟内跑完
- 证据链随机抽查 ≥4 个 URL 可访问且内容相符

---

## Step 10 — CLI + 持久化 + 测试

**目标**：补齐非核心但完整度需要的部分。

---

## Step 11 — 部署与文档

**目标**：容器化部署 + README 完善。

## 风险与节点决策

| 节点 | 触发条件 | 决策 |
|------|---------|------|
| Step 4 结束 | 端到端跑不通 | 优先调 prompts；降级：减少竞品数量 |
| Step 5 结束 | fan-out 并发问题 | 回退串行 + 前端模拟并发动画 |
| Step 6 结束 | HITL interrupt 不稳定 | 降级：只保留 auto 模式 |
| Step 8 结束 | 前端开发耗时过长 | 降级：用 Streamlit 快速替代 |
| Step 9 结束 | Demo 场景质量不达标 | 切换到预录 trace 回放 |
| 任何阶段 | 单 run > 10 分钟 | 减少竞品数（5→3）/ 减少搜索轮次 |
| 任何阶段 | LLM 成本超预算 | 更多 Agent 切到 Haiku / 减少搜索轮次 |
